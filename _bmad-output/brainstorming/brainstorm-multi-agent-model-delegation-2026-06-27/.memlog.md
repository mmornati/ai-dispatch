- (idea by coach) Peer-to-peer agent bidding instead of central orchestrator
- (direction by user) Orchestration must live inside IDE/Copilot/OpenCode ecosystem, not external infra. Non-single orchestrator is OK within that constraint.
- (idea by coach) Two-phase chain: Task Analyzer (decompose) -> Router (fan-out) -> Sub-agents (execute) -> Consolidator (merge)
- (idea by user) Recursive delegation: each agent can autonomously sub-delegate parts of its task to other sub-agents
- (idea by coach) Model pipeline per agent: chain of models optimized for internal sub-tasks (cheap for scaffolding, Sonnet for logic, cheap for docs)
- (idea by coach) DAG scheduler for agent tasks (like Apache Spark) - compile task into directed acyclic graph of sub-agent invocations
- (direction by user) DAG approach must work in CI/CD for fully autonomous event-triggered agents, not just in-IDE
- (idea by coach) Agent configs include trigger field for CI/CD events - same .md files work in IDE and headless CI runners
- (direction by user) Start with SW development use cases (code review, docs sync, meeting prep, onboarding, incident response). Leave other domains for future expansion doc.
- (idea by coach) Eliminate orchestrator model cost - DAG purely config-driven, no LLM call needed for planning
- (idea by user) Shared knowledge base (filesystem artifacts) instead of redundant context passing between agents
- (idea by coach) Reverse hierarchy: agents pull tasks from shared queue, no central scheduler. CI triggers agent directly, agents sub-delegate to each other.
- (technique by coach) started Agent Mirror Protocol (invented)
- (technique by coach) started Inversion of Control Flow (invented)
- (technique by coach) started Constraint Mapping (coach-picked from catalog)
- (insight by coach) Synthesis: Two entry points (local MCP + CI/CD event), same orchestration engine. Start with local (OpenCode agents + MCP server for Copilot app). Agent Mirror + recursive delegation = self-healing teams.
